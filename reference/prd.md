# 기능 명세서: 건축물대장 면적 조회 서비스

---

## 1. 서비스 개요

공인중개사가 주소와 동/호수를 입력하면, 건축물대장 기준 면적(전용/공용/공급)을 즉시 조회할 수 있는 웹 서비스. 매물 등록 시 면적 불일치로 반려되는 문제를 해결하는 것이 목적.

---

## 2. 사용자 플로우

```
[STEP 1] 주소 검색
  → 카카오 우편번호 팝업 열림
  → 사용자가 도로명 또는 지번 주소 선택
  → 시스템이 bcode(법정동코드), jibunAddress 등 자동 추출

[STEP 2] 동/호수 입력
  → 텍스트 입력 필드 2개: "동" (선택), "호" (필수)
  → 예: 동 = "101", 호 = "501"
  → 동이 없는 건물(오피스텔/상가 등)은 동 비워두고 호만 입력

[STEP 3] 조회 버튼 클릭

[STEP 4] 결과 출력
  → 전용면적: 84.99㎡ (25.71평)
  → 공용면적: 29.17㎡ (8.82평)
  → 공급면적: 114.16㎡ (34.53평)
```

---

## 3. API 연동 상세

### 3-1. 카카오 우편번호 서비스 (프론트엔드)

- 호출: `daum.postcode.load()` → JS SDK
- 사용할 응답 필드:

| 필드 | 설명 | 예시 | 용도 |
|---|---|---|---|
| `bcode` | 법정동코드 10자리 | `1168010100` | 시군구코드 + 법정동코드 추출 |
| `jibunAddress` | 지번주소 | `서울 강남구 역삼동 737` | 번지 추출용 백업 |
| `buildingCode` | 건물관리번호 25자리 | `1168010100107370000` | 건축물대장 PK로 직접 사용 가능 |
| `apartment` | 공동주택 여부 | `Y` / `N` | 집합건물 판별 |

- 코드 파싱:

```
bcode = "1168010100"
sigunguCd = bcode.substring(0, 5)   → "11680"
bjdongCd  = bcode.substring(5, 10)  → "10100"
```

- 번지 파싱 (jibunAddress에서 추출):

```
"역삼동 737"    → bun = "0737", ji = "0000"
"역삼동 737-2"  → bun = "0737", ji = "0002"
```

> ※ 건축물대장 API는 번지를 4자리 zero-padding으로 요구할 수 있으므로 API 명세 확인 필요

### 3-2. 건축물대장 API — 전유공용면적 조회 (백엔드, NextJS API Route)

- 엔드포인트: 국토교통부 건축HUB 건축물대장정보 서비스 → 전유공용면적 조회
- 요청 파라미터:

| 파라미터 | 값 | 출처 |
|---|---|---|
| `sigunguCd` | `11680` | 카카오 bcode 앞 5자리 |
| `bjdongCd` | `10100` | 카카오 bcode 뒤 5자리 |
| `bun` | `0737` | jibunAddress에서 파싱 |
| `ji` | `0000` | jibunAddress에서 파싱 |
| `ServiceKey` | 발급받은 키 | 공공데이터포털 |
| `_type` | `json` | JSON 응답 요청 |

- 응답에서 추출할 필드:

| 필드명 (예상) | 의미 | 출력 |
|---|---|---|
| `donNm` | 동명 | 사용자 입력값과 매칭 |
| `hoNm` | 호명 | 사용자 입력값과 매칭 |
| `exclusUseAr` | 전용면적 (㎡) | 표시 |
| `commonUseAr` | 공용면적 (㎡) | 표시 |
| (전용 + 공용) | 공급면적 (㎡) | 계산하여 표시 |

- 매칭 로직:

```
API 응답 배열에서
  → donNm이 사용자 입력 "동"과 일치
  → hoNm이 사용자 입력 "호"와 일치
하는 row를 필터링하여 면적 추출

주의: 한 호실에 전용면적 row + 공용면적 row가
별도로 내려올 수 있음 → 같은 호실의 row를 합산
```

## 5. 에러 처리

| 상황 | 처리 |
|---|---|
| 주소 검색 결과 없음 | "주소를 찾을 수 없습니다" |
| 건축물대장에 해당 건물 없음 | "건축물대장 정보가 없습니다. 주소를 확인해주세요" |
| 동/호 매칭 실패 | "해당 동/호를 찾을 수 없습니다. 입력값을 확인해주세요" |
| API 호출 실패 (타임아웃/서버 오류) | "조회 중 오류가 발생했습니다. 잠시 후 다시 시도해주세요" |
| 공공데이터 API 일일 한도 초과 | "일일 조회 한도를 초과했습니다" |

## 7. 기술 아키텍처

```
[브라우저]
  ├── 카카오 우편번호 SDK (프론트에서 직접 호출)
  └── NextJS 페이지
        │
        ▼
[NextJS API Route: /api/area]
  ├── 카카오 응답값 수신 (bcode, jibunAddress)
  ├── 코드 파싱 (sigunguCd, bjdongCd, bun, ji)
  ├── 건축물대장 API 호출 (서버사이드, API키 보호)
  ├── 응답에서 동/호 매칭 + 면적 추출
  └── 클라이언트에 면적 데이터 반환
```

> 건축물대장 API 키는 프론트에 노출되면 안 되므로 반드시 NextJS API Route(서버사이드)에서 호출합니다.

## 8. 확정된 스펙 요약

| 항목 | 결정 |
|---|---|
| 기술스택 | NextJS |
| 주소 입력 | 카카오 우편번호 서비스 → 코드 추출 |
| 면적 조회 | 공공데이터포털 건축물대장 API (전유공용면적) |
| 지원 건물 | 아파트/오피스텔, 상가/사무실 (집합건물) |
| 동/호 입력 | 사용자 직접 텍스트 입력 |
| 출력 면적 | 전용면적 + 공용면적 + 공급면적(합계) |
